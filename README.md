##  概要

**BeanJournal Personal**は、日常で楽しむスペシャルティコーヒーの記録を視覚的に管理できるWebアプリケーションです。

- **風味を可視化**: 5つの味覚要素をレーダーチャートで表示
- **コレクション性**: 全国で購入したコーヒー豆をカード形式でストック
- **完全プライベート**: ブラウザのLocalStorageで保存、データは端末内のみ

このアプリは、サークル活動で様々なスペシャルティコーヒーを扱う中で「テイスティングノートが記録として残らない」という課題を解決するために開発しました。

##  主な機能

### 1. コーヒー記録の登録・管理
- **基本情報**: 商品名、生産国、農園、品種、風味
- **詳細情報**: プロセス、使用ドリッパー、購入店
- **その他メモ**: 自由記述欄

### 2. 風味の見える化
5段階スライダーで味の特徴を記録し、レーダーチャートで表示：
- 酸味（Acidity）
- 苦味（Bitterness）
- コク（Body）
- 甘味（Sweetness）
- 香り（Aroma）

### 3. お気に入り管理
- お気に入り登録機能で特に気に入った豆を管理
- ワンタップでお気に入りのON/OFF切り替え

### 4. 編集・削除機能
- 登録済みデータの編集が可能
- カスタムモーダルでの削除確認

### 5. レスポンシブデザイン
- スマートフォン、タブレット、デスクトップに対応
- モバイルファーストで最適化されたUI

### 6. ダークテーマUI
- 目に優しいダークモード
- ガラスモーフィズムを採用した洗練されたデザイン
- ゴールドアクセント（#d4af37）

##  使い方

### インストール不要
1. ブラウザで `index.html` を開く
2. すぐに使い始められます

### 基本操作
1. **新規登録**: 右下の「+」ボタンをタップ
2. **情報入力**: 必須項目（商品名、生産国、農園、品種、風味）を入力
3. **風味調整**: スライダーで5つの味覚要素を設定
4. **保存**: 「保存」ボタンで記録完了
5. **編集**: カードの編集ボタン（鉛筆アイコン）から修正可能
6. **削除**: ゴミ箱アイコンで削除（確認ダイアログあり）
7. **お気に入り**: ★アイコンでお気に入り登録

##  データ保存について

### LocalStorageを使用
- データはブラウザのLocalStorageに保存されます
- **カード1枚あたり**: 約500〜600バイト
- **保存可能数**: 約10,000件以上（ブラウザ依存）

### 注意事項
- データは**端末・ブラウザごとに独立**して保存されます
- ブラウザのキャッシュをクリアするとデータが消える可能性があります
- 別の端末やブラウザとの同期はできません

### 将来のアップデート
データのエクスポート/インポート機能、クラウド同期機能は今後のバージョンで実装予定です。

##  技術スタック

### フロントエンド
- **HTML5**: セマンティックHTML
- **CSS3**: 
  - Flexbox レイアウト
  - CSS変数（カスタムプロパティ）
  - ガラスモーフィズムUI
  - レスポンシブデザイン（モバイル・タブレット対応）
- **Vanilla JavaScript (ES6+)**:
  - DOM操作
  - イベント処理（イベント委譲）
  - LocalStorage API
  - 配列メソッド（forEach, filter, find, findIndex, map）

### ライブラリ
- **[Chart.js](https://www.chartjs.org/)**: レーダーチャート描画
- **[Lucide Icons](https://lucide.dev/)**: SVGアイコン
- **[Google Fonts (Outfit)](https://fonts.google.com/)**: フォント

### 特徴
- **フレームワーク不使用**: Vanilla JSで学習を兼ねた開発
- **軽量**: 外部依存最小限
- **オフライン動作**: ネット接続不要（初回読み込み後）

## 開発の経緯

このプロジェクトは以下の目的で開発されています：

1. **実用性**: サークル活動でのテイスティングノート記録の課題解決
2. **学習**: Vanilla JavaScriptによる実践的なWebアプリ開発の練習
3. **プロトタイプ**: 将来のミニSNS版（BeanJournal Social）の基礎

### なぜ「Personal版」なのか
本アプリはLocalStorageを使用した完全プライベート仕様です。  
将来的には、Firebaseを使用したクラウド同期・ソーシャル機能を持つ「Social版」を別プロジェクトとして開発予定です。
